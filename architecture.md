└─────────────────────────────────────────────────────────────┘

Internal Data Flow:

## 线程安全考虑

```
┌─────────────────────────────────────────────────────────────┐
│                      Thread Safety Model                     │
└─────────────────────────────────────────────────────────────┘

Proxy Thread:
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  progressOps()                                      │   │
│  │                                                     │   │
│  │  1. Read active ops list                           │   │
│  │     (single thread, no lock needed)                │   │
│  │                                                     │   │
│  │  2. Call op->progress()                            │   │
│  │     (transport-specific progress)                  │   │
│  │                                                     │   │
│  │  3. Timeout Check                                  │   │
│  │     - Read current time (CLOCK_MONOTONIC)          │   │
│  │     - Compare with lastActiveTime                  │   │
│  │     - Log if timeout detected                      │   │
│  │                                                     │   │
│  │  4. Update lastActiveTime                          │   │
│  │     (only updated by proxy thread)                 │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Notes:                                                     │
│  - timeout fields only accessed by proxy thread             │
│  - no locks needed for timeout detection                    │
│  - lastActiveTime updated atomically not needed            │
│    (single writer, but volatile for visibility)            │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Main Thread:
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  - Posts operations to opsPool                             │
│  - Does NOT access timeout fields directly                 │
│  - Communication via lock-free queue (opsPool)             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 性能开销分析

```
┌─────────────────────────────────────────────────────────────┐
│                     Performance Overhead                     │
└─────────────────────────────────────────────────────────────┘

When Timeout Disabled (NCCL_PROXY_TIMEOUT_MS=0):
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Overhead: 0                                                │
│                                                             │
│  Code Path:                                                 │
│    if (config.enabled) {   // false                        │
│      // skipped entirely                                    │
│    }                                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘

When Timeout Enabled:
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Per-Check Operations:                                      │
│  1. Get current time (clock_gettime)                       │
│     └── ~20-50 ns (vDSO, no syscall)                       │
│                                                             │
│  2. Compare with last check time                           │
│     └── ~1 ns                                               │
│                                                             │
│  3. If check interval elapsed:                             │
│     └── Iterate all active ops (~10-100 ops)               │
│     └── Iterate all subs (~1-8 per op)                     │
│     └── Calculate elapsed time                             │
│     └── Compare with timeout                               │
│     └── Log if needed                                      │
│                                                             │
│  Typical Overhead per Check Interval (1s):                  │
│  └── < 100 microseconds                                    │
│                                                             │
│  As percentage of 1s interval:                             │
│  └── < 0.01%                                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Per-Progress Update:
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Each time progress is made:                                │
│  └── ncclProxyUpdateActiveTime()                           │
│      └── clock_gettime()                                   │
│      └── ~20-50 ns                                         │
│                                                             │
│  Frequency:                                                 │
│  - Once per sub-operation step                             │
│  - Typically every few microseconds to milliseconds        │
│                                                             │
│  Total overhead negligible (< 0.1% of communication time)  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
